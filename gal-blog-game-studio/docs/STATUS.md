# 第一版功能状态

## 已实现

- Story IR 1.0：项目、章节、场景、块、角色/表情、资源、变量、路线、结局、存档、AI 配置、Blog 动作。
- Zod 结构校验与完整跨引用诊断：重复 ID、章节/场景、角色/表情、变量、路线、结局、存档点、AI 配置及缺失资源。
- 资源/角色/表情别名解析，不静默生成坏路径。
- JSON、Markdown、Ren'Py-like、WebGAL、标签式、中文自然语言导入。
- 全类型剧情块表单编辑、局部修改、禁用、排序、复制与场景创建；舞台动作可编辑表情、站位、转场、时长、音量和 transform。
- 角色创建/编辑、表情差分追加、资源注册与别名测试；项目、WebGAL、Terre、Blog Bridge 和变量注册表均有可视化设置。
- StoryPatch、逆向操作、浏览器撤销/重做、历史抽屉与自动保存。
- 月姬式纵向叙事地图：故事自上而下推进、同层分支横向展开；支持自动竖排、自由拖动、开始、公共线、角色故事、场景故事、普通/坏/真结局。
- 作者视图与玩家视图；节点和分支选择、重新连线、分支条件/优先级编辑、隐藏、解锁、已读、重玩语义，以及场景摘要卡。
- ADV 与 NVL/日记模式，可在同一场景中切换。
- 选择项、自由输入及固定选项+自由输入并存；选项既可跳转，也可执行操作后继续当前场景的下一块。
- 输入到 story/blog/ai 的目标标记；发布运行时向博客和 AI provider 转发 `player-input`。
- 内建 Story IR 预览，含舞台状态、分支、输入、Bridge 三态和 AI fallback。
- Story IR → WebGAL 4.6.2 编译。
- WebGAL Terre 官方动画预设随包导出：淡入淡出、方向入场、震动、推近、模糊、电影滤镜与冲击波；`fade` 友好别名会安全映射为官方 `enter` / `exit`。
- BGM 淡入淡出与 WebGAL 0–100 音量编译；全舞台特效通过 `setAnimation` 实际执行。
- WebGAL 原生块逃生口。
- Web ZIP、Story JSON、嵌入清单与资源清单导出。
- Terre 工程写入、Web 导出与编辑预览 WebSocket 适配层。
- Blog Bridge request/reply、origin/source/capability 校验、多允许来源选择、超时与完整动作集；导出游戏会真实暂停 WebGAL、等待三态结果、写回变量并继续或分支。
- AI 语义工具 API、通用 Patch API、编译 API。
- 完整爱丽丝示例：公共线、文章动作、可达的留言三态流程、NVL 日记、实时 AI 点、好感/解锁变量、坏结局与真结局。

## 明确保留接口、尚未完成生产实现

- AI Provider：已有 tool/patch 协议、AI 配置和运行点，但未绑定具体模型或密钥管理服务。
- 二进制素材上传与生成：已有资源注册、缺失提示和 Terre 上传边界；浏览器版未实现对象存储。
- 完整 WebGAL iframe：Terre/共享 loader 适配已实现；线上 Studio 演示使用内建预览，避免把编辑器部署绑定到本地 Terre。
- Blog 表单：Bridge 与三态剧情已实现，真实留言、友链、图片上传 UI/后端应由 gal-blog 提供。
- 多用户协作与云端持久化：领域层可迁移，当前默认是单用户 localStorage + 文件导入导出。
- 逐操作版本 diff UI：已有 Patch、inverse 和快照历史；当前界面显示操作历史，尚未做字段级并排 diff。
- WebGAL 存档深链到任意 block：嵌入清单和 save point 数据已生成，最终恢复需要宿主/引擎扩展消费 blockId。
- 原生资源打包：ZIP 生成资源清单，不自动复制用户尚未提供的二进制文件。
- AI 实时流式执行：已有受约束回合、工具限制和 fallback；provider 接入后再实现流式审核与逐操作动画。

这些项目没有从架构中删除，入口、数据结构或适配接口均已保留。
